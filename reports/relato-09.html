Corrigindo Bug no kw send-patch e Atualizando Documentação

Investigação e correção do bug da issue #1257 no kw send-patch, onde a flag de versão -v posicionada ao final do comando causava falha no git send-email. Inclui também a atualização do texto de ajuda -h defasado em relação à documentação oficial.

1) O Problema · Issue #1257

A Melissa descreveu na issue que o kw send-patch falha quando a flag de versão -v[N] é colocada ao final do comando. Se ela é usada no início, não há problemas. O erro exibido é: 

$ kw send-patch --send --simulate -8 -v8
No patch files specified!

git send-email [<options>] <file|directory>
...

O comportamento esperado seria que -8 -v8 e -v8 -8 fossem equivalentes, uma vez que o commit range e a versão do patch não deveriam ter dependência de ordem para o usuário.

2) Investigando o Código

O primeiro passo foi localizar o arquivo responsável pelo comando. O kw organiza cada subcomando em seu próprio arquivo dentro de src/, então foi direto ao ponto: src/send_patch.sh.

Após ler o código, mapeei o fluxo de processamento dos argumentos:

  1. A função reposition_commit_count_arg move argumentos do tipo -N (somente dígitos) para depois do --.
  2. O kw_parse processa -v8 como a opção -v com argumento 8, armazenando em PATCH_VERSION='-v8'.
  3. No case --, a variável PASS_OPTION_TO_SEND_EMAIL era montada concatenando os argumentos passados e então o PATCH_VERSION ao final: 
    options_values['PASS_OPTION_TO_SEND_EMAIL']="$(str_strip "$pass_option_to_send_email ${options_values['PATCH_VERSION']}")"
  4. Isso resultava em -8 -v8 (versão ao final) sendo passado ao git send-email. O git exige que a flag de versão venha antes do commit range, daí o comando falhava internamente.

Em resumo: a ordem de concatenação na linha 1122 de src/send_patch.sh estava invertida.

3) A Solução

Dentro da função parse_mail_options, era preciso inverter a ordem de concatenação, colocando o PATCH_VERSION antes de $pass_option_to_send_email. Aproveitei também para remover o espaço explícito entre eles (o loop já inseria o separador): 

# src/send_patch.sh — linha 1122
-  options_values['PASS_OPTION_TO_SEND_EMAIL']="$(str_strip "$pass_option_to_send_email ${options_values['PATCH_VERSION']}")"
+  options_values['PASS_OPTION_TO_SEND_EMAIL']="$(str_strip "${options_values['PATCH_VERSION']} $pass_option_to_send_email")"

Com essa mudança, independentemente da posição da flag -v na linha de comando do usuário, a versão sempre será colocada antes do commit range ao construir o comando git send-email internamente.

4) Testes

O kw utiliza o framework shunit2 para os testes de unidade em Bash. Conforme descrito na documentação, é necessário clonar o shunit2 dentro do diretório tests/: 

cd tests
git clone --depth=1 https://github.com/kward/shunit2

4.1) Novo caso de teste em test_mail_parser

Adicionei um novo caso de teste em tests/unit/send_patch_test.sh cobrindo especificamente a combinação -8 -v3 (versão ao final, que era o cenário com bug): 

# version flag must precede commit count so git send-email accepts it
parse_mail_options -8 -v3

expected='-v3'
assert_equals_helper 'Set version with commit count: PATCH_VERSION' \
  "$LINENO" "$expected" "${options_values['PATCH_VERSION']}"

expected='-v3 -8'
assert_equals_helper 'Set version with commit count: PASS_OPTION_TO_SEND_EMAIL' \
  "$LINENO" "$expected" "${options_values['PASS_OPTION_TO_SEND_EMAIL']}"

assert_equals_helper 'Set version with commit count: COMMIT_RANGE' \
  "$LINENO" '-8 ' "${options_values['COMMIT_RANGE']}"

4.2) Atualizando test_email_send

O teste test_email_send existente também precisou ser atualizado para refletir a nova ordem correta dos parâmetros: 

# antes
expected='git send-email --to="mail@test.com" extra_args --other_arg -13 -v2'

# depois
expected='git send-email --to="mail@test.com" -v2 extra_args --other_arg -13'

5) Verificação Manual

Para garantir que a correção funcionava de ponta a ponta, primeiro reproduzi o erro original com o kw instalado sem as mudanças: 

$ kw send-patch --send --simulate -8 -v8
No patch files specified!

git send-email [<options>] <file|directory>
...

Em seguida, instalei o kw localmente com as alterações:

cd ~/kworkflow   # fork com as alterações
./setup.sh --install

E rodei o comando novamente com --verbose para inspecionar o git send-email gerado internamente: 

$ kw send-patch --send --simulate --verbose -8 -v8
git send-email --dry-run --annotate --cover-letter --no-chain-reply-to --thread -v8 -8

A flag -v8 agora precede o commit range -8, exatamente como o git send-email exige. O bug está corrigido.

7) Atualizando a Documentação -h

Aproveitando que já estava modificando o src/send_patch.sh, atualizei o texto de ajuda exibido pelo kw send-patch -h, que estava defasado em relação à documentação oficial. A versão anterior não documentava várias flags que já existiam no comando: 

-  'send-patch (-s | --send) [<options>] - Send patches via e-mail' \
+  'send-patch (-s | --send) [--simulate] [--private] [--rfc] [--to=<recipient>] [--cc=<recipient>] [<rev-range>] [-v<n>] [-- <extra>] - Send patches via e-mail' \

-  'send-patch (-t | --setup) [--local | --global] [-f | --force] (<config> <value>)...' \
+  'send-patch (-t | --setup) [--local | --global] [-f | --force] (<config> <value>)... - Configure send-email options' \

-  'send-patch (-i | --interactive) - Setup interactively' \
+  'send-patch (-i | --interactive) [--local | --global] - Setup interactively' \

-  'send-patch --verify - Check if required configurations are set' \
+  'send-patch --verify [--local | --global] - Check if required configurations are set' \

-  'send-patch --template[=<template>] [-n] - Set send-email configs based on <template>' \
+  'send-patch --template[=<template>] [-n | --no-interactive] [--local | --global] [-f | --force] - Set send-email configs based on <template>' \

Criando um commit separado para essa mudança:

git add src/send_patch.sh
git commit
git push origin unstable

Com dois commits na branch unstable do fork, o Pull Request foi aberto no repositório oficial e está disponível em #1265, aguardando a revisão do David.
